Scout is built around one abstraction: every source is a `ContextProvider`. Six providers ship in the box. Any MCP server plugs in. Custom providers are a four-method subclass.

Each new source follows the same pattern:

1. Configure auth (env var, OAuth, service account, MCP token).
2. Register the provider in [`scout/contexts.py`](https://github.com/agno-agi/scout/blob/main/scout/contexts.py).
3. Restart Scout.
4. Verify with `curl http://localhost:8000/contexts | jq`.

Each provider becomes `query_<id>` (and optionally `update_<id>`) tools on Scout. The agent calls them like any other tool.

## Connect Google Drive

Scout reads Drive as **its own service-account identity**, not yours. You create the account once with the `./scripts/google_setup.sh` helper, share folders with the service account email, and set `GOOGLE_SERVICE_ACCOUNT_FILE` in `.env`.

Why a service account? Borrowing your credentials would collapse audit trails and mean everything Scout does shows up as something *you* did. Its own identity keeps actions attributable.

Full setup, including the manual path for orgs that block service-account keys: [GDRIVE_CONNECT.md](https://github.com/agno-agi/scout/blob/main/docs/GDRIVE_CONNECT.md).

## Connect any MCP server

Linear, GitHub, Notion, Postgres, Stripe, anything with an [MCP server](https://modelcontextprotocol.io) plugs in by adding an `MCPContextProvider` entry to [`scout/contexts.py`](https://github.com/agno-agi/scout/blob/main/scout/contexts.py):

```python
MCPContextProvider(
    server_name="linear",
    transport="stdio",
    command="npx",
    args=["-y", "@linear/mcp"],
    env={"LINEAR_API_KEY": getenv("LINEAR_API_KEY", "")},
    model=default_model(),
)
```

Add the secret to `.env`, restart, and Scout gets a `query_mcp_linear` tool.

The full reference, including the choice between `mode=tools` (flatten the server's tools onto Scout) and `mode=default` (wrap in a sub-agent), and the gotcha around Node-based servers in Docker: [MCP_CONNECT.md](https://github.com/agno-agi/scout/blob/main/docs/MCP_CONNECT.md).

## Build your own provider

If your source isn't covered by an MCP server, write a `ContextProvider`. Four methods: `status`, `astatus`, `query`, `aquery`. Subclass, register in `scout/contexts.py`, done.

The Scout README has the worked example in its [Add your own](https://github.com/agno-agi/scout#add-your-own) section. The [`agno.context.web.provider`](https://github.com/agno-agi/agno/blob/main/libs/agno/agno/context/web/provider.py) module is a complete reference implementation.
