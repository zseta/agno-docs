The Agno team provides a number of starter templates for running AgentOS, and agentic applications built on top of it. Each template is a pre-configured git repo you clone. There are two kinds of templates:

1. **AgentOS Starters:** clean AgentOS installations, one per platform: Railway, Docker, AWS, Fly.io, and Google Cloud Run.
2. **Application Blueprints:** complete agentic products that you can deploy for internal use.

## What's inside every Template

Every starter template comes with a common platform core.

| Component | What it does |
|-----------|--------------|
| **Two platform agents** | Agent Builder creates agents, teams, and workflows. Platform Manager helps manage the platform. |
| **Two workflows** | _Deployment Check_ checks platform health and _Run Evals_ runs the eval suite to catch regressions. |
| **MCP server** | Makes your platform available to any AI app or MCP client. |
| **Coding-agent skills** | Manage the full agent development lifecycle using skills like `/create-new-agent`, `/improve-agent`, `/eval-and-improve`, and more. |
| **Eval suite** | Full AgentAsJudge and Reliability Evals. |
| **PostgreSQL** | Sessions, memory, knowledge, and traces stored in your database. |
