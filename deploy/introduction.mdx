The Agno team provides a number of starter templates for running AgentOS, and agentic applications built on top of it. Each template is a pre-configured git repo you clone. There are two kinds of templates:

1. **AgentOS Starters:** clean AgentOS installations, one per cloud: Railway, AWS, GCP, Azure.
2. **Application Blueprints:** complete agentic products that you can deploy for internal use.

## What's inside every Template

Every starter template comes with a similar platform core. Only the deploy layer differs.

| Component | What it does |
|-----------|--------------|
| **Agent Builder** | Platform agent. Creates agents, teams, and workflows from chat using the [AgentOS Studio](/agent-os/studio/introduction). |
| **Platform Manager** | Platform agent. Read-only ops: eval history, deployment checks, schedules, codebase questions. |
| **MCP server** | `/mcp` endpoint. Any MCP client can run your agents, teams, and workflows. |
| **Coding-agent skills** | `/create-new-agent`, `/improve-agent`, `/eval-and-improve`, and more in `.agents/skills/`. |
| **Eval suite** | Judge and reliability cases. Results are logged to Postgres and show up at os.agno.com. |
| **PostgreSQL + pgvector** | Sessions, memory, knowledge, and traces stay in your database. |

  <Step title="Set up with a coding agent">
    Every Starter leads with a copy-paste prompt. Paste it into Claude Code, Cursor, or Codex; the agent handles the clone, credentials, Docker, and verification. Each Starter page also lists the manual steps; App pages walk through setup directly.
  </Step>
  <Step title="Connect your frontends">
    Run `uvx agno connect` to register the platform with the coding agents on your machine, or connect the [AgentOS UI](https://os.agno.com). Once deployed to production, re-run `uvx agno connect --url https://<your-domain>` to point your coding agents at production, and register `https://<your-domain>/mcp` as a remote MCP server in Claude and ChatGPT; they sign in with OAuth.
