We provide pre-built templates for running AgentOS in your cloud. Every template is a GitHub repo you clone and own, with everything wired up:

1. **The codebase.** Agents, evals, an MCP server, and coding-agent skills, ready to build on.
2. **The infrastructure.** Containers, PostgreSQL, and deploy scripts for your platform of choice.
3. **The setup.** Copy one prompt into your coding agent and it takes you from clone to a running platform.

There are two kinds. **Starters** are clean AgentOS installations, one per platform: Railway, Docker, AWS, Fly.io, or Google Cloud Run. Start with one when you're building your own agents. **Apps** are complete agent products (Dash, Coda, Scout, and Context) that you deploy and make your own.

## One backend, every frontend

AgentOS is a secure, scalable backend for building agents once and making them available everywhere:

1. **AI apps.** Claude and ChatGPT can use your agents through the MCP server at `/mcp`.
2. **Chat interfaces.** Chat with your agents from Slack, WhatsApp, Telegram, and Discord.
3. **Coding agents.** Let Claude Code and Codex run your platform using the skills in `.agents/skills/`.
4. **Your product.** Embed agents directly into your product with the AgentOS REST API: 80+ endpoints for runs, sessions, memory, knowledge, evals, and more.
5. **AgentOS UI.** Chat with agents, inspect sessions, traces, memory, and evals at [os.agno.com](https://os.agno.com).

## What's inside every Starter

Every Starter ships the same platform core. Only the deploy layer differs.

| Component | What it does |
|-----------|--------------|
| **Agent Builder** | Platform agent. Creates agents, teams, and workflows from chat using the [AgentOS Studio](/agent-os/studio/introduction). |
| **Platform Manager** | Platform agent. Read-only ops: eval history, deployment checks, schedules, codebase questions. |
| **WebSearch** | Reference agent. The simplest pattern to copy when building your own. |
| **MCP server** | `/mcp` endpoint. Any MCP client can run your agents, teams, and workflows. |
| **Coding-agent skills** | `/create-new-agent`, `/improve-agent`, `/eval-and-improve`, and more in `.agents/skills/`. |
| **Eval suite** | Judge and reliability cases. Results log to Postgres, visible at os.agno.com. |
| **PostgreSQL + pgvector** | Sessions, memory, knowledge, and traces stay in your database. |
