**AgentOS is the agent backend for every frontend. Templates are complete codebases that take you from clone to a running platform in minutes.**

Every template is a GitHub repo you clone and own. There are two kinds:

| Kind | The job | Pick one when |
|------|---------|---------------|
| **Starters** | Where AgentOS runs. A clean platform wired for one target: Railway, Docker, AWS, Fly.io, or Google Cloud Run. | You're building your own agents. |
| **Apps** | What the agent does. A complete agent product: Dash, Coda, Scout, or Context. | You want a working product to deploy and adapt. |

## One backend, every frontend

Your platform serves every surface at once:

| Surface | How it connects |
|---------|-----------------|
| Your product | REST API with 80+ endpoints: runs, sessions, memory, knowledge, evals. |
| Chat interfaces | Slack is wired in. WhatsApp, Telegram, and Discord follow the same pattern. |
| AI apps | Claude and ChatGPT drive your platform through the MCP server at `/mcp`. |
| Coding agents | Claude Code, Codex, and Cursor build and improve the platform with bundled skills. |
| AgentOS UI | [os.agno.com](https://os.agno.com): chat with agents, inspect sessions, traces, memory, and evals. |

## What's inside every Starter

Every Starter ships the same platform core. Only the deploy layer differs.

| Component | What it does |
|-----------|--------------|
| **Agent Builder** | Flagship agent. Creates agents, teams, and workflows from chat using the AgentOS Studio. |
| **Platform Manager** | Flagship agent. Read-only ops: eval history, deployment checks, schedules, codebase questions. |
| **WebSearch** | Reference agent. The simplest pattern to copy when building your own. |
| **MCP server** | `/mcp` endpoint. Any MCP client can run your agents, teams, and workflows. |
| **Coding-agent skills** | `/create-new-agent`, `/improve-agent`, `/eval-and-improve`, and more in `.agents/skills/`. |
| **Eval suite** | Judge and reliability cases. Results log to Postgres, visible at os.agno.com. |
| **PostgreSQL + pgvector** | Sessions, memory, knowledge, and traces stay in your database. |
