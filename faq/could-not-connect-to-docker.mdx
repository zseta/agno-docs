If Docker is up and running but you get the following error, read on:

```bash
ERROR    Could not connect to docker. Please confirm docker is installed and running
ERROR    Error while fetching server API version: ('Connection aborted.', FileNotFoundError(2, 'No such file or directory'))
```

## Quick fix

Create the `/var/run/docker.sock` symlink:

```shell
sudo ln -s "$HOME/.docker/run/docker.sock" /var/run/docker.sock
```

This fixes most cases. If it doesn't, fix the socket's ownership:

```shell
sudo chown $USER /var/run/docker.sock
```

## Full details

The `agno-infra` package uses [docker-py](https://github.com/docker/docker-py) to run containers. If `/var/run/docker.sock` is missing or has incorrect permissions, docker-py cannot connect to the Docker daemon.

**To fix, create the `/var/run/docker.sock` symlink:**

```shell
sudo ln -s "$HOME/.docker/run/docker.sock" /var/run/docker.sock
```

If that does not work, check the permissions with `ls -l /var/run/docker.sock`.

If `/var/run/docker.sock` does not exist, check whether `$HOME/.docker/run/docker.sock` is also missing. If it's missing, reinstall Docker.

**If none of this works and `/var/run/docker.sock` exists:**

- Give your user permissions to the `/var/run/docker.sock` file:

```shell
sudo chown $USER /var/run/docker.sock
```

- Add your user to the docker group:

```shell
sudo usermod -a -G docker $USER
```
