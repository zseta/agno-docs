Every team building agents builds the same system from scratch:

- A server to run the agents (batch, streaming, or background mode).
- A database for sessions, runs, traces, and memory.
- Auth validated via JWT on every request.

This system is the agent platform. Build the foundation once so every new agent slots into the same runtime, storage, and connectors.

This section walks through the reference Starter, [agentos-railway](https://github.com/agno-agi/agentos-railway): AgentOS on FastAPI with Postgres. The other eight [Starters](/deploy/templates) share the same core and swap the deploy layer.

## What the Starter ships

| Surface | Contents |
| ------- | -------- |
| `agents/` | **Agent Builder** creates agents, teams, and workflows at runtime. **Platform Manager** monitors and explains the platform. **WebSearch** answers from live web results with citations. |
| `workflows/` | `deployment-check` verifies platform wiring and readiness. `run-evals` runs the eval suite. |
| `app/registry.py` | The tools, models, databases, functions, and agents Agent Builder can wire into new components. |
| `app/schedules.py` | Cron schedules registered at startup. The deployment check runs daily by default; scheduled evals are opt-in. |
| `evals/` | Regression cases run with `python -m evals`. |
| `.agents/skills/` | Coding-agent skills covering the lifecycle: create, extend, improve, evaluate, review. |
| `scripts/railway/` | Deploy, env-sync, redeploy, and teardown scripts for Railway. |

## Managed by coding agents

Agent code, logs, traces, evals, and the live service live in one repo. Claude Code manages the full lifecycle with the skills in `.agents/skills/`: create an agent, improve it, extend it, and evaluate it.
