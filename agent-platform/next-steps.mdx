## Beyond single agents

| Pattern | Use it when | Docs |
| ------- | ----------- | ---- |
| **Agent** | A single model with tools and instructions can handle the request | [Agents](/agents/overview) |
| **Team** | The right specialist is not known up front; a leader routes or coordinates | [Teams](/teams/overview) |
| **Workflow** | The process must run the same way every time | [Workflows](/workflows/overview) |

Agent Builder creates all three as Studio components from chat. For the code path, `workflows/` ships two reference workflows to copy from: `deployment-check` and `run-evals`.

## Scheduled tasks

The scheduler is on in `app/main.py` (`scheduler=True`). `app/schedules.py` registers crons at startup:

| Workflow | Default | What it does |
| -------- | ------- | ------------ |
| `deployment-check` | Daily cron on | Deterministic readiness report: DB, auth, scheduler URL, MCP, Slack config. No model calls. |
| `run-evals` | Cron off | Runs the `EVALS_TAG` cases and returns a report. Enable with `ENABLE_SCHEDULED_EVALS=True`. |

To add your own, define a workflow in `workflows/`, register it in `AgentOS(workflows=[...])`, and add a cron in `register_schedules()`. See [scheduling](/features/scheduling).

## Connect interfaces

Slack is wired in `app/main.py` and activates when `SLACK_BOT_TOKEN` and `SLACK_SIGNING_SECRET` are set:

```python app/main.py
from agno.os.interfaces.slack import Slack

interfaces: list = []
if SLACK_BOT_TOKEN and SLACK_SIGNING_SECRET:
    interfaces.append(
        Slack(
            agent=agent_builder,
            streaming=True,
            token=SLACK_BOT_TOKEN,
            signing_secret=SLACK_SIGNING_SECRET,
            resolve_user_identity=True,
        )
    )
```

## Use the platform from MCP clients

`enable_mcp_server=True` in `app/main.py` mounts an MCP server at `/mcp`. Claude, ChatGPT, Cursor, and Claude Code can run your agents, teams, and workflows through tools like `run_agent`. Register the endpoint with the coding agents on your machine:

```bash
uvx agno connect
```

See the [AgentOS MCP server](/agent-os/mcp/mcp).

## Keep the repo coherent

As agents accumulate, configuration drifts: env vars go stale and new agents miss registration. Run the maintenance skill before a release or after a refactor:

```
/review-and-improve
```

It diffs docs against code, auto-fixes mechanical drift (stale paths, missing `example.env` entries, agents on disk missing from `app/main.py`), and surfaces the rest as a punch list.
