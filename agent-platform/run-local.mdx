The reference Starter runs two containers:

- **agentos-api**: AgentOS on FastAPI, port 8000.
- **agentos-db**: Postgres with PgVector, port 5432.

Stateless FastAPI in front of Postgres. The same stack you run in production.

## Prerequisites

- [Docker Desktop](https://docs.docker.com/desktop/) installed and running
- An [OpenAI API key](https://platform.openai.com/api-keys) for models and embeddings

## Run your agent platform

<Steps>
  <Step title="Clone the Starter">
    ```bash
    git clone https://github.com/agno-agi/agentos-railway.git agentos
    cd agentos
    ```

    <Tip>
    To make this codebase yours, run `rm -rf .git` and push to your own git repo.
    </Tip>
  </Step>

  <Step title="Configure your environment">
    ```bash
    cp example.env .env
    ```

    Open `.env` and set `OPENAI_API_KEY`. Everything else has working defaults. Optional keys:

    | Key | Purpose |
    | --- | ------- |
    | `PARALLEL_API_KEY` | Authenticates the WebSearch agent's Parallel connection. Without it, the agent falls back to Parallel's keyless MCP server. |
    | `SLACK_BOT_TOKEN` + `SLACK_SIGNING_SECRET` | Set both to expose Agent Builder on Slack. |
    | `ENABLE_SCHEDULED_EVALS` | `True` schedules a daily eval run. Off by default because it uses model calls. |
  </Step>

  <Step title="Start the platform">
    ```bash
    docker compose up -d --build
    ```

    Compose starts the FastAPI app on port 8000 and Postgres on port 5432. It sets `RUNTIME_ENV=dev`, which turns JWT auth off for local development. Production requires JWT auth; see [Run on Railway](/agent-platform/run-railway).
  </Step>

  <Step title="Verify it's running">
    Open [http://localhost:8000/docs](http://localhost:8000/docs).

    You'll see the OpenAPI spec: runs, sessions, memory, knowledge, and evals exposed as REST endpoints.

    <Frame caption="AgentOS API">
      <img src="/videos/first-agent-api.gif" alt="AgentOS API" />
    </Frame>
  </Step>
</Steps>

AgentOS also comes with a UI at [os.agno.com](https://os.agno.com).

## Connect the AgentOS UI

1. Open [os.agno.com](https://os.agno.com) and log in.
2. Click **Connect OS**, choose **Local**, enter `http://localhost:8000` as the URL, and name it **Local AgentOS**.
3. Click **Connect**.

You should see three agents:

| Agent | What it does |
| ----- | ------------ |
| **Agent Builder** | Creates agents, teams, and workflows at runtime from the registry in `app/registry.py`. Components are versioned; deletes pause for your approval. |
| **Platform Manager** | Read-only monitor. Answers codebase questions, reports eval history, runs deployment checks, and lists schedules and runtime-built components. |
| **WebSearch** | Searches the web with Parallel tools and answers with cited sources. |

Try a prompt against each:

> _"Build an agent that tracks AI news and writes a daily brief"_ → Agent Builder creates and publishes the agent. Click **Refresh** and it appears in the Agents list.

> _"How healthy is the platform?"_ → Platform Manager answers from deployment checks, eval history, and schedules.

> _"What's the latest research from Anthropic?"_ → WebSearch returns a cited summary.

Open **Sessions** and **Traces** in the sidebar. Every run is captured with full message history, tool calls, and timing. Sessions and traces power the improve loop in [Improve an Agent](/agent-platform/improve-agent).

## What else is running

| Surface | Details |
| ------- | ------- |
| Workflows | `deployment-check` verifies DB, auth, scheduler, and component wiring. `run-evals` runs the eval suite. Both are REST endpoints and visible in the UI. |
| Schedules | Registered at startup. The deployment check runs daily by default (`ENABLE_DEPLOY_CHECK`); scheduled evals are opt-in (`ENABLE_SCHEDULED_EVALS`). |
| MCP server | Any MCP client can call your agents through `/mcp`. Run `uvx agno connect` to register `http://localhost:8000/mcp` with Claude Code, Claude Desktop, Codex, and Cursor. |

<Tip>
  Hot-reload is on: uvicorn watches `agents/`, `app/`, `db/`, `evals/`, and `workflows/`. Edits go live in about 2 seconds.
</Tip>
