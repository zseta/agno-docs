Two skills in `.agents/skills/` iterate on an existing agent:

| Skill | Who drives | Use it to |
| ----- | ---------- | --------- |
| `/improve-agent` | Claude | Harden the agent against its own `INSTRUCTIONS` |
| `/extend-agent` | You | Add a tool or capability, refine instructions, fix a known bug |

Both edit `agents/<slug>.py` against the live container. Compose runs uvicorn with a scoped `--reload`, so each edit is live in a couple of seconds.

## Harden with /improve-agent

Open Claude Code in your `agentos` directory and run:

```
/improve-agent
```

Claude asks for the target slug, reads the agent's `INSTRUCTIONS`, and derives 8-12 probes across four categories: golden path, edge cases, tool selection, and adversarial. For each probe it calls the live agent over REST, reads which tools fired from the container logs, and judges PASS or FAIL against what the instructions promise.

For every failure, Claude picks one lever per iteration:

| Lever | When |
| ----- | ---- |
| `INSTRUCTIONS` | Most fixes. Tighten or add a rule. |
| Tools | Add a missing tool, or remove a misused one. |
| Context provider | Swap mode when the routing layer is the problem. |
| `num_history_runs` | Raise if context is lost across turns; lower if old turns leak in. |
| Model | Bump only if the agent is genuinely under-capable. Last resort. |

Claude re-runs only the failed probes, spot-checks one or two passing probes for regressions, and caps at five iterations. The run ends with a diff and a suggested commit message.

<Warning>
Probes against `agent-builder` create real Studio components: its create, edit, and publish tools execute immediately. The skill brackets the loop with `snapshot_component_ids()` and `delete_new_components()` from `evals/cases.py`, so probe runs never leave components behind.
</Warning>

## Change with /extend-agent

When you have a specific change in mind, run:

```
/extend-agent
```

Claude reads the agent, restates its purpose, and asks what to change: add a tool, add a capability (knowledge base, memory, sub-agent, scheduled task), refine instructions, or fix a bug. Toolkit research goes through the agno-docs MCP, so import paths and constructor args match the real API. Each iteration is one change, smoke-tested against the live container before the next.

## When to run each

| Situation | Run |
| --------- | --- |
| Just created an agent and want to harden it before deploying | `/improve-agent` |
| Users report the agent misses the point | `/improve-agent` |
| You want a new tool or knowledge base | `/extend-agent` |
| You hit a specific bug | `/extend-agent` |
| You just extended an agent and want to catch regressions | `/improve-agent` |
