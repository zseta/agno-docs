title: Human Approval
description: "Pause runs for user confirmation, admin approval, or external execution."
---

Some agent actions shouldn't run unattended: sending a refund, restarting a production service, deleting a customer record, posting to a public Slack channel. AgentOS lets you mark a tool as needing human approval, and the run pauses until that human responds.

Approval can come from three places:

| Approver | Pattern | Example |
|----------|---------|---------|
| **The user** asking the agent | `requires_confirmation=True` | "Approve this refund I just asked for?" |
| **An admin** with policy authority | `@approval(type="required")` | "Refunds over $500 need a finance lead's sign-off" |
| **An external system** with its own approval flow | `external_execution=True` | "Submit to our existing change-management tool" |

```python
from agno.tools import tool
from agno.approval.decorator import approval

@approval(type="required")
@tool(requires_confirmation=True)
def issue_refund(customer_id: str, amount: float, reason: str) -> str:
    return charge_refund(customer_id, amount)

agent = Agent(model="openai:gpt-5.4", tools=[issue_refund])
```

When the agent decides to call `issue_refund`, the run pauses. The API surfaces a pending approval. A human approves or rejects. The run resumes with the answer.

## The four pause patterns

| Pattern | Decorator / param | Behavior |
|---------|-------------------|----------|
| **User confirmation** | `@tool(requires_confirmation=True)` | "Do you approve this exact tool call?" Yes/No. |
| **User input** | `@tool(requires_user_input=True)` | "I need more info to proceed." Free-form answer. |
| **External execution** | `@tool(external_execution=True)` | "Run this externally and tell me the result." For systems with their own approval flows. |
| **Admin approval** | `@approval(type="required" \| "audit")` | Policy gate independent of the user. `required` blocks. `audit` logs without blocking. |

`@approval` and `requires_confirmation` compose. The audit trail captures both the tool call and every human decision along the way.

## User confirmation

The user asking the agent to do something gets to confirm the actual tool call before it fires:

```python
from agno.tools import tool

@tool(requires_confirmation=True)
def restart_service(service_name: str) -> str:
    return ops_client.restart(service_name)
```

In Slack, the agent posts an inline confirmation prompt. In the AgentOS UI, the run shows up as **Awaiting approval** with Approve / Reject buttons. The user clicks. The run resumes.

This is the right pattern for "this exact action, yes or no?" cases. Reversibility doesn't matter. Even reversible actions benefit from "are you sure?" before executing.

## Admin approval

Some actions need approval from someone with *policy authority*, not just the user asking. Refunds over a threshold. Data exports for compliance review. Production deploys.

```python
from agno.approval.decorator import approval
from agno.tools import tool

@approval(type="required")
@tool
def export_customer_data(customer_id: str) -> str:
    return get_customer_data(customer_id)

@approval(type="audit")
@tool
def generate_compliance_report(month: str) -> str:
    return build_report(month)
```

| `@approval(type=...)` | Behavior |
|-----------------------|----------|
| `"required"` | Run blocks until a designated approver signs off. The audit trail captures the approver's identity. |
| `"audit"` | Run continues. Tool call gets logged to the audit trail asynchronously. Used when policy says "track but don't gate." |

Admin approval routes to the configured approver pool, not the user asking. See [Approvals](/agent-os/approvals/overview) for routing setup.

## User input mid-run

Sometimes the agent needs more info from the user before it can finish. `requires_user_input=True` pauses and asks:

```python
@tool(requires_user_input=True)
def file_support_ticket(summary: str) -> str:
    # The agent is told it must collect a description from the user
    # before this tool can complete.
    ...
```

The agent prompts the user with the structured request. The user responds. The run resumes with the response in scope.

For more elaborate forms (multi-question, branching), use `UserFeedbackTools` and `UserControlFlowTools`. See the [Feedback agent demo](/demo-os/human-in-the-loop).

## External execution

Some tools route through systems that have their own approval flows. A change-management tool. A Jira workflow. A CI pipeline that requires manual promotion.

```python
@tool(external_execution=True)
def submit_to_change_management(payload: dict) -> str:
    # AgentOS marks the run as awaiting external execution.
    # Your external system processes it, then calls back to AgentOS
    # with the result.
    return submit(payload)
```

The agent doesn't wait on a Python call to return. It hands off, the run pauses, the external system reports back via the AgentOS API, the run resumes.

## Resuming a paused run

The agent returns a `RunOutput` with `active_requirements`. Inspect them, decide, continue:

```python
run = agent.run("Refund customer ACME-123 for $500")

for req in run.active_requirements:
    if req.needs_confirmation:
        print(f"Tool: {req.tool_name}, args: {req.tool_args}")
        req.confirm()  # or req.reject(reason="...")

result = agent.continue_run(run_id=run.run_id, requirements=run.requirements)
```

In the AgentOS UI and Slack, this happens via buttons. The same `continue_run` endpoint is what those surfaces call.

## Guardrails for inputs

Pre-hooks run before the model sees the user's message. Use them to mask PII, block prompt injections, or moderate content:

```python
from agno.guardrails import PIIDetectionGuardrail, PromptInjectionGuardrail

agent = Agent(
    model="openai:gpt-5.4",
    pre_hooks=[
        PIIDetectionGuardrail(
            mask_pii=True,
            enable_ssn_check=True,
            enable_credit_card_check=True,
            enable_email_check=True,
            enable_phone_check=True,
        ),
        PromptInjectionGuardrail(),
    ],
)
```

The user's input gets sanitized before it reaches the model: PII is masked and injection attempts are blocked. The agent never sees the raw input if a guardrail rejects it.

## Hooks for outputs

Post-hooks run after the model produces output. Use them for output guardrails, audit logs, notifications:

```python
from agno.hooks import hook

@hook(run_in_background=True)
def audit_log(run_output, agent):
    log_to_audit_table(
        agent_id=agent.id,
        user_id=run_output.user_id,
        content=run_output.content,
        tools_used=[t.name for t in run_output.tool_calls],
    )

agent = Agent(model=..., post_hooks=[audit_log])
```

`run_in_background=True` makes the hook run as a FastAPI background task so the user gets the response without waiting on the audit write.

## Picking the right pattern

| Situation | Use |
|-----------|-----|
| Tool call is irreversible (refund, delete, deploy) | `requires_confirmation=True` |
| Action needs sign-off from someone other than the user | `@approval(type="required")` |
| Action needs to be logged for compliance, not gated | `@approval(type="audit")` |
| Agent needs more info from the user mid-run | `requires_user_input=True` |
| Tool runs in another system with its own approval flow | `external_execution=True` |
| Block PII / prompt injection before the model sees it | `pre_hooks=[PIIDetectionGuardrail(), PromptInjectionGuardrail()]` |
| Audit every run | `post_hooks=[audit_hook]` with `run_in_background=True` |
