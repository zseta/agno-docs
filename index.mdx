**Agno turns agents into production software.**

Build your agent in any framework. Run it as a service with AgentOS. Ship it to real users.

<CodeGroup>

```python Agno SDK
from agno.agent import Agent
from agno.db.sqlite import SqliteDb
from agno.os import AgentOS
from agno.tools.hackernews import HackerNewsTools

agent = Agent(
    name="Agno Agent",
    model="openai:gpt-5.4",
    tools=[HackerNewsTools()],
)

agent_os = AgentOS(
    agents=[agent],
    tracing=True,
    db=SqliteDb(db_file="tmp/agentos.db"),
)
app = agent_os.get_app()

if __name__ == "__main__":
    agent_os.serve(app="agno_agent:app", reload=True)
```

```python LangGraph
from agno.agents.langgraph import LangGraphAgent
from agno.db.sqlite import SqliteDb
from agno.os import AgentOS
from langchain_openai import ChatOpenAI
from langgraph.graph import MessagesState, StateGraph

def chatbot(state: MessagesState):
    return {"messages": [ChatOpenAI(model="gpt-5.4").invoke(state["messages"])]}

graph = StateGraph(MessagesState)
graph.add_node("chatbot", chatbot)
graph.set_entry_point("chatbot")
compiled = graph.compile()

agent = LangGraphAgent(
    name="LangGraph Chatbot",
    graph=compiled,
)

agent_os = AgentOS(
    agents=[agent],
    tracing=True,
    db=SqliteDb(db_file="tmp/agentos.db"),
)
app = agent_os.get_app()

if __name__ == "__main__":
    agent_os.serve(app="langgraph_agent:app", reload=True)
```

AgentOS gives you a production runtime with one import:

- **Production API.** 50+ endpoints with SSE and websockets to build your product on.
- **Context & state.** Sessions, memory, knowledge, and traces stored in your Postgres.
- **Human-in-the-loop.** Pause agent execution and wait for user, admin, or external approval.
- **Scheduled workflows.** Run agents on a schedule and trigger background jobs.
- **Interfaces.** Expose your agent through Slack, Discord, Telegram, or a custom UI.
- **Observability.** Native tracing and full audit logs for every run.
- **Security and auth.** JWT-based authentication with hierarchical scopes and access controls.
- **Request isolation.** No state bleed between users, agents, or sessions.
