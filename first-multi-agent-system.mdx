## What we're building today

A multi-agent system built on [`agno-agi/agentos-railway`](https://github.com/agno-agi/agentos-railway), the reference AgentOS Starter. It ships with three agents:

| Agent | What it does |
|-------|--------------|
| **Agent Builder** | Creates agents, teams, and workflows through the [AgentOS Studio](/agent-os/studio/introduction). |
| **Platform Manager** | Read-only ops: eval history, deployment checks, schedules, codebase questions. |
| **WebSearch** | Searches the web with Parallel. The simplest agent to copy. |

You'll run it locally with Docker, then deploy to Railway. Prerequisites:

- [Docker Desktop](https://www.docker.com/products/docker-desktop)
- [OpenAI API key](https://platform.openai.com/api-keys)

Prefer a coding agent to drive the setup? [AgentOS on Railway](/deploy/templates/railway/deploy) has a copy-paste prompt for Claude Code, Cursor, or Codex.

## Run locally

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/agno-agi/agentos-railway.git agentos
    cd agentos
    ```
  </Step>
  <Step title="Configure environment">
    ```bash
    cp example.env .env
    ```
    Add your `OPENAI_API_KEY` to `.env`.
  </Step>
  <Step title="Start the application">
    ```bash
    docker compose up -d --build
    ```
    The first build takes a few minutes. Confirm the API at [localhost:8000/docs](http://localhost:8000/docs).
  </Step>
  <Step title="Verify end to end">
    ```bash
    ./scripts/mcp_check.sh
    ```
    Prints `MCP OK` with the tool count and a real agent answer through the MCP endpoint.
  </Step>
  <Step title="Connect the AgentOS UI">
    1. Open [os.agno.com](https://os.agno.com)
    2. Click **Connect OS**, enter `http://localhost:8000`, and name it **Local AgentOS**
  </Step>
</Steps>

Your multi-agent system is now running at `localhost:8000`.

## Try your agents

### Agent Builder

Agent Builder creates agents, teams, and workflows from chat using the AgentOS Studio. Try:

- Build an agent that tracks AI news and writes a daily brief

Once it's built, click **Refresh** in the top right, pick the new agent from the **Agents** dropdown, and ask: "What's new with Anthropic?"

### Platform Manager

Platform Manager answers from the codebase and runtime data. Try:

- How healthy is the platform?
- What evals ran recently?

### WebSearch

WebSearch answers questions with live web results and is the simplest agent to copy when building your own. Try:

- What's the latest news on AI agents?

## Deploy to production

<Steps>
  <Step title="Install Railway CLI">
    <CodeGroup>
    ```bash Mac
    brew install railway
    ```
    ```bash Windows
    npm install -g @railway/cli
    ```
    </CodeGroup>
    Then run `railway login`.
  </Step>
  <Step title="Create a production env">
    ```bash
    cp .env .env.production
    ```
    Edit `.env.production` with production values: a different OpenAI key, production-only credentials.
  </Step>
  <Step title="Deploy">
    ```bash
    ./scripts/railway/up.sh
    ```
    The script provisions PostgreSQL and the `agent-os` service on the same private network, then pauses for a JWT verification key.
  </Step>
  <Step title="Mint your JWT key">
    1. Open [os.agno.com](https://os.agno.com), click **Connect OS** → **Live**, enter your Railway domain, and name it **Live AgentOS**
    2. Go to **Settings** → **OS & Security** and turn on **Token-Based Authorization (JWT)**
    3. Copy the public key and paste the full PEM into the `up.sh` prompt

    <Note>Live AgentOS connections are a paid feature. Use code `PLATFORM30` for one month off.</Note>
  </Step>
  <Step title="Confirm it's live">
    ```bash
    railway logs --service agent-os
    ```
    Open `https://<your-domain>/docs` to confirm the API is serving.
  </Step>
</Steps>

Your multi-agent system is now live.

## Manage your deployment

```bash
railway logs --service agent-os     # Tail logs
railway open                        # Open the dashboard
./scripts/railway/redeploy.sh       # Deploy code changes
```
